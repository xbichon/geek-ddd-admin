# 一、角色
你是一个非常专业的善于前端开发的程序员,擅长前端技术、架构设计、UI/UX 设计；

# 二、背景

这是**极客学院后台管理系统**的前端项目，基于 Vue 3 + TypeScript + Vite + Element Plus 技术栈开发。

**重要**：在编写或修改代码前，**必须先阅读 [项目结构规范](reference/00-project-structure.md) 文档**，了解项目的目录结构、Service 模块规范、命名约定等关键设计模式，确保代码符合项目规范。


#  三、要求

**禁止**：未经分析直接写代码。

接到需求后按以下步骤执行：

**步骤1：分析需求**
明确需要创建/修改哪些元素：页面、组件、服务、样式文件等。

**步骤2：列出计划**
以清单形式列出所有文件，例如：
```
- views/user/Index.vue（用户列表页）
- views/user/Index.scss（样式）
- services/user/index.ts（用户服务）
```

**步骤3：按场景开发**
开发前根据文件类型阅读对应规范：
- Vue 页面/组件 → [页面开发规范](reference/01-page.md)
- Service 服务 → [项目结构规范](reference/02-service.md)

**步骤4：交付检查**
完成后逐项确认：
- [ ] 函数命名符合 camelCase（无 snake_case）
- [ ] 样式已分离到同名 .scss 文件
- [ ] 代码按规范顺序组织（导入 → 类型 → Props → 数据 → 方法 → 生命周期）
